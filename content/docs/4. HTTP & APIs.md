## 4. HTTP & APIs

---

### 22. HTTP 401 — Unauthorized

- Means **authentication required or failed**
- Triggered when:
    - No API key provided
    - Invalid or expired API key
    - Wrong credentials
- ❌ NOT "resource not found" (that's 404)
- ❌ NOT "server error" (that's 500)
- ❌ NOT "rate limit exceeded" (that's 429)

---

### 23. HTTP 429 — Too Many Requests

- Means **rate limit exceeded**
- API allows only a fixed number of requests per minute/hour
- Example: OpenWeatherMap free tier = 60 calls/minute
- Fix: add `time.sleep(1)` or `time.sleep(2)` between requests
- ❌ NOT "internal server error" (that's 500)
- ❌ NOT "unauthorized" (that's 401)

---

### 24. HTTP 3xx — Redirection

- Status codes **300 to 399** indicate redirection
- Correct check:

```python
300 <= status < 400  # ✅ correct

status in [301, 302, 303]  # ❌ incomplete
200 < status < 400         # ❌ includes success codes too
status in range(299, 400)  # ❌ starts from 299, wrong
```

- Common codes:
    - 301 → Moved Permanently
    - 302 → Found (Temporary Redirect)
    - 304 → Not Modified

---

### 25. HTTP 2xx — Success

- Status codes **200 to 299** indicate success
- Correct check:

```python
200 <= status < 300  # ✅ correct

status == 200               # ❌ misses 201, 202 etc.
status in [200, 201, 202]   # ❌ incomplete
status < 400                # ❌ includes redirects too
```

- Common codes:
    - 200 → OK
    - 201 → Created
    - 204 → No Content

---

### 26. HTTP GET — Safe Method

- **Safe** = does NOT modify server state
- Only retrieves/reads data
- Can be cached and repeated without side effects
- Used for: fetching data, reading resources
- ❌ POST → creates/sends data (not safe)
- ❌ PUT → updates data (not safe)
- ❌ DELETE → removes data (not safe)

---

### 27. HTTP PUT — Idempotent Method

- **Idempotent** = multiple identical requests have same effect as single request
- Sending PUT 5 times = same result as sending once
- Used for: updating/replacing a resource
- Other idempotent methods: GET, DELETE
- ❌ POST is NOT idempotent (creates new resource each time)

---

### 28. HTTP POST — Sending Data

- Used for:
    - Submitting data to server
    - **LLM API inference requests**
    - Creating new resources
- Request body contains the data (e.g., review text for sentiment analysis)
- NOT idempotent — multiple POSTs create multiple resources
- ❌ GET → only retrieves, cannot send body data
- ❌ DELETE → removes resources
- ❌ PUT → updates existing resources

---

### 29. API Rate Limiting — Handling

- Problem: API allows limited calls per minute (e.g., 60/min)
- Solution: add delay between requests

```python
import time
import requests

cities = ['Delhi', 'Mumbai', 'Chennai']
for city in cities:
    response = requests.get(url, params={'q': city})
    process(response)
    time.sleep(1)  # ✅ 1-2 sec delay between calls
```

- ❌ Making all requests simultaneously → triggers 429 errors
- ❌ Ignoring rate limit → API blocks your key
- ❌ Making requests only at off-peak hours → not reliable

---

### 30. API Security — Storing API Keys

- ✅ Store in **environment variables**

```python
import os
api_key = os.getenv('WEATHER_API_KEY')  # ✅ correct
```

- ❌ Hard-coding in script:

```python
api_key = "abc123xyz"  # ❌ never do this
```

- ❌ Writing in code comments → visible to everyone
- ❌ Including in GitHub README → publicly exposed
- Why? Hard-coded keys get exposed in version control history

### 31. JSON Parsing — Nested Fields

- API responses are often nested JSON structures
- Example response:

```json
{
  "weather": [
    {"description": "clear sky"}
  ],
  "main": {"temp": 298.15}
}
```

- Correct access:

```python
response_json['weather'][0]['description']  # ✅ correct
# weather is a LIST → need index [0] first

response['weather']['description']    # ❌ missing list index
response['main']['weather']           # ❌ wrong key
response_json['description']['weather'] # ❌ wrong order
```

---

### 32. `requests` / `httpx` — Python HTTP Library

- Most commonly used Python library for HTTP requests
- Basic usage:

```python
import requests

response = requests.get(url, params={'q': 'Delhi', 'appid': api_key})
data = response.json()  # parse JSON response
```

- ❌ `pandas` → for data manipulation, not HTTP requests
- ❌ `flask` → web framework, not HTTP client
- ❌ `beautifulsoup` → HTML parsing, not HTTP requests

---

### 33. Web Scraping Ethics — `robots.txt`

- `robots.txt` file tells scrapers which paths are allowed/disallowed
- ✅ Ethical scraping:
    - Parse `robots.txt` file before scraping
    - Follow disallowed paths
    - Implement request delays (`crawl-delay` directive)
    - Respect rate limits
- ❌ Ignoring `robots.txt` even for academic purposes → unethical
- ❌ Treating `robots.txt` as optional guidance → incorrect
- Tools: `robotparser` module in Python
